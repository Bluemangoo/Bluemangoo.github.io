Windows 11 子系统(WSA)安装方法

之前安装了 Windows Subsystem for Android™ ，顺手写篇教程吧。

系统环境

Windows 11

下载WSA

打开这个网址：https://store.rg-adguard.net/

左侧选择 ProductId ，右侧选择 Slow

中间的产品ID填 9P3395VX91NR

然后点 “√” 开始解析。

解析完找到最下面那个（一个G以上的），下载下来。

安装WSA

按 Win+X ，打开管理员权限的 Windows Terminal

Shift+右键下载下来的文件，选择”复制为路径”

在wt里面输入：

等待安装完成就行了。

运行WSA

安装完成在开始菜单中运行Windows Subsystem for Android™ Settings

点第一栏文件右侧的箭头

如果成功打开，那么恭喜你，安装完成了！（请跳转到下一步）

如果报错，请打开appwiz.cpl（直接在开始菜单搜索就行）

打开左侧打开或关闭Windows功能

找到Hyper-V，勾上，点确定，完成后重启电脑。

这里注意一点，如果安装了非Hyper-V版本的VMware的话，需要卸载VMware。VMware 15 及以上支持安装Hyper-V版本，请在安装VMware的时候注意下。

如果Hyper-V这个复选框无法点击，请重启进入Bios设置并开启虚拟化。

不同主板的菜单不一样，如果不会请自行百度。
Bios里面找到带有virtual的基本上就是了。

再设置完成再次尝试打开Hyper-V

然后重启，再次尝试打开Windows Subsystem for Android™ Settings中的文件

*再失败我也救不了你。*

配置WSA

下载 adb 工具 下载地址

将adb解压并添加到Path

建议解压到一个好找的地方，比如D:\\adb

在Windows Subsystem for Android™ Settings里面打开开发者模式。

打开一次文件并关闭。

打开Windows Terminal并输入：

等待连接成功。

接下来就可以使用adb了！

安装应用

建议下载一个应用市场，举个例子：

  • 豌豆荚
  • 酷安
  • APKPure

使用以下命令安装apk（需要先连接WSA）

建议cd到下载目录安装。

然后就可以愉快的使用WSA了。

显示网络受限的处理办法

由于谷歌的验证服务器被墙，需要改为国内服务器。

改完重启下WiFi就行
